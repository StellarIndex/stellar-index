# Findings Register

This register starts empty. Findings are added during workstream
execution and during the mandatory passes. Severity per
[11-severity-rubric.md](11-severity-rubric.md).

## How to add a finding

1. Pick the next unused ID (zero-padded `F-####`, starting at
   `F-1201`). The 12 prefix denotes 2026-05-12 audit.
2. Capture supporting evidence in the appropriate ledger first,
   then copy the IDs into the finding row. Per protocol §4 the
   ledgers + prefixes are: `EV` (general), `CMD` (commands),
   `XFI` (cross-file), `J` (journey), `R1` (R1 probe).
3. If the finding came from an adversarial pass, link the
   attack-tree row in [10-attack-tree.md](10-attack-tree.md).
4. Set disposition. Default `open` if real and triaged; use
   `needs_evidence` if you suspect but can't yet prove; use
   `needs_owner` if confirmed but the remediation plan hasn't
   assigned a wave yet. See protocol §12 for the full taxonomy.
5. Update the remediation plan
   ([07-remediation-plan.md](07-remediation-plan.md)) at the
   same time so the finding is sequenced.
