# Remediation Plan

This file remains mostly empty until findings are created. Do not add
remediation items that are not tied to cold-audit findings or explicit
notes.

## Prioritization Rules

1. Critical and high findings that affect market-data correctness,
   security, customer trust, or live operations come first.
2. Medium findings that can silently corrupt data outrank obvious
   visible failures.
3. Docs fixes follow code fixes unless the doc defect itself creates
   immediate operator/customer risk.
4. Competitive product gaps are prioritized by customer-facing impact,
   not implementation ease.
