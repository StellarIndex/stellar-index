# Audit — 2026-05-26 — Cold, Adversarial, Pre-Launch

This directory holds the full audit of the Rates Engine repository
and live R1 deployment, executed as a **cold, adversarial** review.

- **Cold:** prior audit findings, closure summaries, CLAUDE.md
  claims, ADR text, runbook prose, CHANGELOG entries, commit
  messages, in-code docstrings, prior auditors' notes — **none of
  these are accepted as evidence**. Every claim is re-tested
  against current code, current tests, or live R1 runtime.
- **Adversarial:** the frame is "if someone hostile (attacker,
  competitor, journalist, sloppy operator, abusive customer) wants
  to break, mis-trust, exfiltrate, exhaust, or embarrass this
  system, how would they do it?" See [10-attack-tree.md](10-attack-tree.md).
- **Pre-launch:** the system is live on R1 but no public consumer
  traffic flows yet. This is the last cheap window to find
  correctness, security, and contract bugs before the public flip.

## Why this audit is bigger than the 2026-05-12 one

Since 2026-05-12 (commit `bb37e1e9` baseline), the codebase added:

- **17 new migrations** (0029–0045), including the soroban_events
  raw landing zone (0041, ADR-0029) and four decoder backfill
  tables (0042–0045).
- **5 new source packages**: `cctp`, `defindex`, `rozo`,
  `sorobanevents`, `soroswap_router`.
- **1 new external source**: `chainlink`.
- **3 new ADRs**: 0027 (LCM cache tiering), 0028 (RWA asset
  representation), 0029 (soroban_events landing zone).
- **6 new operator subcommands**: `cctp-backfill`, `rozo-backfill`,
  `soroswap-skim-backfill`, `comet-liquidity-backfill`,
  `phoenix-backfill`, `blend-backfill`.
- **Cross-cutting correctness changes**: back-pressure semantics
  on the soroban_events sink (rc.80); `TolerateTrailingMissing`
  for bounded walks (rc.81); per-chunk resume + Type=notify for
  verify-archive; fd-2 wrap drain-on-exit (rc.77/rc.78); ON
  CONFLICT shape match for migration 0041 (rc.79).
- **Live ops experience that changed our trust posture**: ZFS
  pool exhaustion incident (2026-05-17), MinIO credential drift,
  cursor-stuck diagnostics, prewarm/handler arg drift, fd-2 wrap
  silent-drop on short-lived processes, AsyncSink cursor-drop
  incoherence (cost ~18.86M rows on the 2026-05-26 fill walk).

The baseline audit had 26 workstreams. This audit has **35**, with
nine additions explicitly addressing the deltas above and one
expanded scope on the "every event" granularity mission (the
project-level commitment to having no partial-event decoders).

## Headline findings (as of 2026-05-26 22:50 UTC execution iteration 3)

**The cascade chain** — the audit's single most consequential
finding cluster:

```
F-0001 root partition `/` 100% full (49G/49G)
       caused by F-0006 unrotated postgres log 11G + F-0008 va
       residue 4GB + F-0009 ratesengine log 504M + F-0010
       operator artefacts 1.9GB

  → F-0041 Redis persistence dir is `/var/lib/redis` (on root)

  → F-0039 Redis MISCONF — BGSAVE keeps failing on full disk →
       `stop-writes-on-bgsave-error` kicks in → all SET/HSET
       commands disabled (since at least 17:20 UTC, 5+ hours)

  → F-0036 Counters fail .Inc() on Redis write → series never
       appear in Prometheus TSDB

  → F-0027 / F-0032 / F-0033 Alert pipeline structurally
       silent on `rate() == 0` checks because the counter
       series doesn't exist (no series → no eval → no alert)

  → Customer-facing API silently serves stale data (28h+ old
       per F-0043) with `stale:true` flag (positive: API IS
       transparent about it per F-0042)
```

**Pre-launch verdict**: production has been in this degraded
state for at least 5 hours (likely longer per Redis last-save
timestamp). Alertmanager has 0 active alerts beyond the
deadmansswitch heartbeat. The current alerting design CANNOT
detect this class of failure because the metrics signal a
silent failure mode.

**Other launch-blocking findings**:
- F-0018: DeFindex partial-event decoder (harvest/rebalance/
  create/n_fee out of scope despite project_every_event_principle)
- F-0030: CoinGecko free-tier 429 hit (1h backoff)
- F-0034: /v1/diagnostics/cursors public exposure of operational
  state

See [05-findings-register.md](05-findings-register.md) for the
full register (44 findings; 2 critical, 14 high, 10 medium,
6 low, 3 invalid, 3 accepted).

## The Bar

This is not a "good enough" review. The closure rule:

> A reviewer should be able to open this directory cold, read
> README → 00-plan → 01-tracker, follow every claim through
> evidence, and re-walk every workstream in their own session
> without asking the original auditor a single question.

If any of those five steps requires a personal hand-off, the audit
is not done.
