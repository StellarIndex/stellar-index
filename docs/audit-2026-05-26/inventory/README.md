# Inventory

Repo + runtime inventory artefacts. Most are generated by
[`generate.sh`](generate.sh) from the current working tree so a
reviewer can regenerate the snapshot from a fresh clone.

| File | What it contains |
| --- | --- |
| [generate.sh](generate.sh) | regenerates all files in this directory |
| [repo-snapshot.md](repo-snapshot.md) | commit SHA, dirty-tree state, top-level dir listing |
| [area-counts.md](area-counts.md) | file counts per area (cmd, internal, web, etc.) |
| [file-coverage.tsv](file-coverage.tsv) | one row per tracked file; workstream owner + status |
| [adr-inventory.md](adr-inventory.md) | ADR-0001..0029 list + status |
| [api-route-inventory.md](api-route-inventory.md) | every registered route with method + auth gate |
| [alert-rule-inventory.md](alert-rule-inventory.md) | every Prometheus rule + multi-host/R1 pairing |
| [dependency-inventory.md](dependency-inventory.md) | direct deps + transitive surface |
| [docker-systemd-inventory.md](docker-systemd-inventory.md) | Dockerfiles + systemd units |
| [external-source-inventory.md](external-source-inventory.md) | external adapter list + class |
| [metric-name-inventory.md](metric-name-inventory.md) | every metric emitted vs documented |
| [migration-inventory.md](migration-inventory.md) | every migration + reader correspondence |
| [runbook-inventory.md](runbook-inventory.md) | runbook ↔ alert pairing |
| [source-decoder-inventory.md](source-decoder-inventory.md) | on-chain source decoder list |
| [webfrontend-inventory.md](webfrontend-inventory.md) | web/explorer + web/dashboard + web/status |
| [workflow-inventory.md](workflow-inventory.md) | GitHub Actions workflow list + triggers |
| [every-event-coverage.tsv](every-event-coverage.tsv) | W35 register — one row per (source, contract, on-chain-event) |

## Discipline

`file-coverage.tsv` columns:
```
file_path	file_kind	workstream	status	evidence_refs	notes
```
Statuses: `todo` / `in_progress` / `done` / `blocked` / `excluded`.

Run `bash generate.sh` whenever the repo state changes
materially during the audit; never hand-edit the generated rows.
